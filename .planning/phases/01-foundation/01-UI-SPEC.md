# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Phase 1 delivers infrastructure only: Navbar, Footer, and the global CSS token layer.
> No portfolio sections are styled in this phase — those ship in Phase 2.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — no component library in Phase 1 |
| Preset | not applicable |
| Component library | none (shadcn/ui may be evaluated in Phase 2) |
| Icon library | lucide-react (tree-shakable, stroke-based, zero runtime dependencies) |
| Body font | Inter — loaded via `next/font/google`, variable weight |
| Code font | JetBrains Mono — loaded via `next/font/google`, weight 400 + 600 |

**Font rationale (per D-08):** Inter + JetBrains Mono are placeholder selections for Phase 1.
The final brand typeface pair is locked during the Phase 2 design pass. The token names
(`--font-sans`, `--font-mono`) are stable — only the font-family values change in Phase 2.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, badge padding |
| sm | 8px | p-2 / gap-2 | Inline element spacing, nav link gaps |
| md | 16px | p-4 / gap-4 | Default element padding |
| lg | 24px | p-6 / gap-6 | Section inner padding |
| xl | 32px | p-8 / gap-8 | Layout column gaps |
| 2xl | 48px | py-12 | Major section vertical rhythm |
| 3xl | 64px | py-16 | Page-level top/bottom spacing |

Exceptions:
- Navbar height: **56px** (`h-14`) — not on the 8-point grid but standard for fixed nav bars; provides adequate touch target clearance
- Social icon touch target: **44px × 44px minimum** (`min-w-11 min-h-11`) — WCAG 2.5.5 AAA target size

## Typography

All sizes are the final rendered pixel values. Tailwind class equivalents listed for executor reference.

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Body | 16px | `text-base` | 400 (regular) | 1.6 | Paragraph text, footer copyright |
| Label | 14px | `text-sm` | 600 (semibold) | 1.4 | Nav links, caption text, metadata |
| Heading | 24px | `text-2xl` | 600 (semibold) | 1.25 | Section headings (Phase 2+) |
| Display | 36px | `text-4xl` | 600 (semibold) | 1.15 | Hero name (Phase 2+) — token declared here for global.css |

**Phase 1 active roles:** Label (nav links) and Body (footer text) only.
Heading and Display are declared as CSS tokens now so Phase 2 inherits them without rework.

**Font stack (CSS variables):**
```

--font-mono: 'JetBrains Mono', 'Fira Code', ui-monospace, monospace;
```

## Color

Dark-only palette. Light mode vars are not defined in Phase 1 — they ship in Phase 5 (PLSH-01).
All values are CSS custom properties in `globals.css` under `:root` (dark is the only mode).

| Role | CSS Var | Hex | Usage |
|------|---------|-----|-------|
| Dominant (60%) | `--color-bg` | `#0a0a0a` | Page background (`<body>` bg) |
| Secondary (30%) | `--color-surface` | `#111111` | Navbar bg, Footer bg, future card surfaces |
| Foreground | `--color-text` | `#e4e4e7` | Body text, nav link text |
| Muted | `--color-text-muted` | `#71717a` | Copyright line, secondary metadata |
| Border | `--color-border` | `#27272a` | Navbar bottom border, Footer top border, future card borders |
| Accent (10%) | `--color-accent` | `#22d3ee` | Specific elements listed below |
| Accent hover | `--color-accent-hover` | `#06b6d4` | Accent elements on hover |
| Destructive | `--color-destructive` | `#ef4444` | Not used in Phase 1 — declared for completeness |

**Accent (`#22d3ee` cyan-400) is reserved for:**
1. Nav link active/hover state underline or text color change
2. Social icon links in Footer on hover
3. External link hover state (Phase 2+)
4. Inline code text color within prose (Phase 3+)

Accent is NOT used for: general body text, nav link default state, background fills, decorative borders.

**Tailwind config mapping** (`tailwind.config.ts` extension):
```
colors: {
  bg: '#0a0a0a',
  surface: '#111111',
  text: { DEFAULT: '#e4e4e7', muted: '#71717a' },
  border: '#27272a',
  accent: { DEFAULT: '#22d3ee', hover: '#06b6d4' },
  destructive: '#ef4444',
}
```

## Navbar Contract

**Height:** 56px (`h-14`)
**Position:** `fixed top-0 left-0 right-0` (sticky overlay)
**Background:** `--color-surface` (`#111111`) with `border-b` using `--color-border`
**Backdrop:** `backdrop-blur-sm` to bleed depth behind the nav — acceptable in dark mode without visible artifacts

**Layout (desktop only — no hamburger in Phase 1):**
```
[Logo/Name]                    [Home]  [Blog]
```
Left: Site name "Yash Kadam" in Label (14px / 600) styled as a home link (`href="/"`)
Right: Two nav links — "Home" (`href="/"`) and "Blog" (`href="/blog"`) — Label size, 600 (semibold) weight
Gap between nav links: `gap-6` (24px)

**Nav link states:**
- Default: `--color-text` (`#e4e4e7`)
- Hover: `--color-accent` (`#22d3ee`) — color transition, `transition-colors duration-150`
- Active/current page: `--color-accent` with a 2px bottom border in accent color

**Body offset:** Root layout applies `pt-14` to `<main>` to clear the fixed navbar.

## Footer Contract

**Position:** `static` at the bottom of each page (not sticky/fixed)
**Background:** `--color-surface` (`#111111`) with `border-t` using `--color-border`
**Padding:** `py-8` (32px vertical), `px-6` (24px horizontal) on mobile; centered max-width container

**Layout:**
```
Yash Kadam                     [GitHub] [LinkedIn] [X]
© 2025 Yash Kadam
```
Single row on desktop: name left, icons right.
Two rows on mobile: name + copyright stacked left, icons right.

**Name:** "Yash Kadam" — Body (16px / 400) in `--color-text`
**Copyright:** "© 2025 Yash Kadam" — Label (14px / 400) in `--color-text-muted`
**Social icons:** lucide-react icons at 20px (`size={20}`)
- GitHub: `Github` icon — `href="https://github.com/yash-278"` — `target="_blank" rel="noopener noreferrer"`
- LinkedIn: `Linkedin` icon — `href="https://linkedin.com/in/yashkadam"` (executor uses correct URL)
- X/Twitter: `Twitter` icon — `href="https://twitter.com/yashkadam278"` — `target="_blank" rel="noopener noreferrer"`

**Social icon states:**
- Default: `--color-text-muted` (`#71717a`)
- Hover: `--color-accent` (`#22d3ee`) — `transition-colors duration-150`

**Touch target:** Icon wrapper uses `p-2` to reach 44×44px minimum.

**No destructive actions in Phase 1.**

## Accessibility Contract

- Navbar landmark: `<nav aria-label="Main navigation">`
- Footer landmark: `<footer>`
- Skip link: Visually hidden `<a href="#main-content">Skip to main content</a>` as first focusable element in `<body>` — becomes visible on focus with accent color
- Social icons: Each icon link has `aria-label` (see Copywriting Contract above); icons are `aria-hidden="true"`
- Focus ring: `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-accent` on all interactive elements
- Color contrast: `#e4e4e7` on `#0a0a0a` = 15.3:1 (exceeds WCAG AAA 7:1) — `#e4e4e7` on `#111111` = 13.5:1 (exceeds AAA)
- Accent `#22d3ee` on `#0a0a0a` = 8.2:1 (exceeds AAA) — safe for accent text and icons

## Animation Contract

Phase 1 wires Framer Motion (`LazyMotion` + `domAnimation`) but does NOT add entrance animations to Navbar or Footer. They render statically.

**Rule:** No `motion.*` imports in `components/Navbar.tsx` or `components/Footer.tsx` in Phase 1.
**Rule:** `LazyMotion` wraps the root layout with `features={domAnimation}` — executor must confirm `domAnimation` is loaded lazily to avoid adding to the initial bundle.

First scroll animations ship in Phase 2 (portfolio sections).

## Implementation Notes for Executor

1. **CSS variables in `globals.css`:** Declare all `--color-*` and `--font-*` tokens under `:root`. No `@media (prefers-color-scheme: dark)` block — dark is the only mode.
2. **`tailwind.config.ts` darkMode:** Set `darkMode: 'class'` — the `dark` class on `<html>` is added by the inline blocking script in `layout.tsx`.
3. **Inline blocking script pattern:**
   ```html
   <script dangerouslySetInnerHTML={{ __html: "document.documentElement.classList.add('dark')" }} />
   ```
   This must be the first child of `<head>`, before any stylesheet links, and must NOT have `defer` or `async`.
4. **Font variables:** Load Inter and JetBrains Mono via `next/font/google`. Expose as CSS variables (`variable: '--font-sans'` and `variable: '--font-mono'`). Apply both variable classes to `<html>`.
5. **Container:** Use a `<div className="max-w-5xl mx-auto px-6">` wrapper inside both Navbar and Footer — not on the nav/footer element itself — so the background spans full width while content is constrained.
6. **`cn()` utility:** Port `cn()` (clsx + tailwind-merge) from `origin/v2:lib/utils.ts` — it will be needed for conditional class merging on nav active states.
