<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MDX frontmatter strings → RSS XML output | Post titles and descriptions are developer-authored but must be XML-escaped before injection |
| VERCEL_ENV → crawl permission | robots.txt allow/disallow depends on correct environment variable injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03 | Tampering | app/feed.xml/route.ts XML output | mitigate | escapeXml() helper required for all user-controlled strings (title, description) before XML injection; prevents malformed XML |
| T-04-04 | Information Disclosure | app/robots.ts on staging | mitigate | `VERCEL_ENV === 'production'` check ensures staging deployments return Disallow: /, preventing indexing of preview builds |
| T-04-05 | Elevation of Privilege | app/sitemap.ts listing unpublished posts | accept | getAllPosts() already filters unpublished posts (data.published flag); sitemap only lists what getAllPosts() returns |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; all APIs are Next.js built-ins and Next.js server utilities |
</threat_model>
