<objective>
Blog posts are shareable on social media with a unique branded OG image per post. This plan wires three closely related pieces: (1) the root layout gets `metadataBase` so OG URLs resolve correctly everywhere, (2) `/blog/[slug]` page gets full `openGraph` and `twitter` metadata fields, and (3) a new `opengraph-image.tsx` file generates a dark-themed PNG with the post title at build time.

Per D-01: dark card with `#0a0a0a`/`#111` gradient background.
Per D-02: post title + `yashkadam.com` site name only (no avatar, no tags).
Per D-03: static `public/og.png` fallback inherited from root layout metadataBase for all non-post pages.
Per D-04: `app/blog/[slug]/opengraph-image.tsx` using `next/og` ImageResponse, statically generated via `generateStaticParams`.
Per D-13: `generateMetadata` on `/blog/[slug]` gets OG fields with absolute URLs.

Purpose: Enables social-sharing cards with per-post identity, a key Phase 4 success criterion.
Output: `opengraph-image.tsx` (new), `app/blog/[slug]/page.tsx` (modified), `app/layout.tsx` (modified).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MDX frontmatter → OG image title | Post titles are developer-authored MDX content; not user-submitted, but still displayed in generated images |
| Environment variables → metadata URLs | NEXT_PUBLIC_SITE_URL is a build-time env var; wrong value produces wrong sitemap/OG URLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | opengraph-image.tsx | accept | Title comes from developer-authored MDX frontmatter, not user input; no PII risk |
| T-04-02 | Tampering | metadataBase URL | mitigate | Use `process.env.NEXT_PUBLIC_SITE_URL ?? 'https://yashkadam.com'` fallback to prevent `undefined` in metadata URLs if env var is missing |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; all APIs are Next.js built-ins |
</threat_model>
