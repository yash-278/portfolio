## Color Contract

**Accent change in Phase 2:** The Phase 1 accent `#22d3ee` (cyan-400) is replaced with `#818cf8` (indigo-400).
Rationale: Indigo is distinctive in the dev portfolio space, less common than teal/cyan, and reads as
considered taste at dark luminance levels. Pairs cleanly with `#0a0a0a` at 8.1:1 contrast (exceeds WCAG AA).

Update `globals.css` `:root` block:
```css
--color-accent: #818cf8;        /* indigo-400 — replaces #22d3ee */
--color-accent-hover: #6366f1;  /* indigo-500 — hover state */
```

All existing `text-accent` and `hover:text-accent` utility classes in Navbar and Footer automatically
pick up the new values — no component edits needed for the color swap.

| Role | CSS Var | Hex | Usage |
|------|---------|-----|-------|
| Dominant (60%) | `--color-bg` | `#0a0a0a` | Page background |
| Secondary (30%) | `--color-surface` | `#111111` | Navbar, Footer, card base |
| Foreground | `--color-text` | `#e4e4e7` | All body text |
| Muted | `--color-text-muted` | `#71717a` | Dates, captions, role metadata |
| Border | `--color-border` | `#27272a` | Structural dividers, card borders |
| Accent (10%) | `--color-accent` | `#818cf8` | Specific elements — see below |
| Accent hover | `--color-accent-hover` | `#6366f1` | Accent on pointer hover |
| Destructive | `--color-destructive` | `#ef4444` | Not used in Phase 2 |
| Glow / overlay | n/a | `#818cf8` at 6% opacity | Hero background radial glow only |

**Accent (`#818cf8` indigo-400) is reserved for:**
1. Hero CTA link ("View work" + "GitHub") — text color on hover
2. Active nav link state (underline + text color)
3. Social icon links in Footer + Contact section on hover
4. Tech stack tag text color
5. Work history company name text
6. Project card "Live" and "GitHub" link text on hover

Accent is NOT used for: body text, section headings, card backgrounds, decorative borders,
hero name, hero pitch line, photo borders.

## Background Treatment

**Hero section glow:**
Apply a single centered radial gradient behind the hero split layout:
```css
background-image: radial-gradient(ellipse 80% 50% at 50% -10%, rgba(129, 140, 248, 0.06) 0%, transparent 70%);
```
This is applied as an inline `style` or CSS class on the hero section element — not in `globals.css`.
The glow is subtle (6% opacity) and vanishes outside the hero viewport — it does not persist across sections.

**No noise texture.** Noise adds render cost and CSS complexity for marginal visual gain at this opacity level.

**All other sections:** Plain `--color-bg` (`#0a0a0a`) background — no gradients, no patterns.

## Animation Contract

**Approach:** Scroll-triggered entrance animations via Framer Motion `m.*` components inside
`'use client'` leaf components. Each section gets its own Client Component wrapper for animation
— the section content itself remains Server Component rendered markup.

**Animation primitive (apply to every section block):**
```tsx
// SectionReveal.tsx — 'use client'
import { m } from 'framer-motion'

const variants = {
  hidden: { opacity: 0, y: 24 },
  visible: { opacity: 1, y: 0 },
}

<m.div
  variants={variants}
  initial="hidden"
  whileInView="visible"
  viewport={{ once: true, margin: '-80px' }}
  transition={{ duration: 0.5, ease: 'easeOut' }}
>
  {children}
</m.div>
```

**Stagger rule:** When multiple items appear in a list (work history rows, project cards),
wrap each item in `SectionReveal` with a `transition={{ delay: index * 0.1 }}`.
Maximum delay: 0.3s (3 items × 0.1s) — never exceed 0.4s total stagger window.

**Hero exception:** Hero content uses entrance animation on mount (not scroll):
```tsx
initial="hidden"
animate="visible"
// No whileInView — fires immediately on load
```
Name, title, pitch, and CTA each stagger at 0.1s increments.
Photo entrance: same `opacity: 0 → 1, y: 24 → 0` with 0.3s delay.

**Hard constraints (from ROADMAP.md Phase 5 criteria):**
- Only `opacity` + `y` translate — no scale, rotate, or blur
- No `layout` prop on scroll-heavy components
- `viewport={{ once: true }}` on all `whileInView` animations — plays once only
- No section triggers more than one animation per viewport entry
