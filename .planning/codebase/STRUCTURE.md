## Directory Purposes

**`src/`:**
- Purpose: All hand-authored source files
- Contains: CSS source (`index.css`), JS behaviour (`index.js`), static assets
- Key files: `src/index.css`, `src/index.js`

**`src/assets/`:**
- Purpose: Static images and SVG illustrations referenced by `index.html`
- Contains: `.jpg` photos and `.svg` hero illustrations
- Key files: `src/assets/img3.jpg` (used in About section `<img src="./src/assets/img3.jpg">`)

**`dist/`:**
- Purpose: Compiled CSS output consumed by the browser
- Contains: `main.css` only — generated by Tailwind CLI from `src/index.css`
- Generated: Yes
- Committed: Yes (currently tracked in git — see CONCERNS.md)

**`.planning/codebase/`:**
- Purpose: GSD architecture/stack/convention documents
- Contains: Markdown analysis files
- Generated: By GSD mapping commands
- Committed: Optional (per team convention)

## Key File Locations

**Entry Points:**
- `index.html`: Browser entry point — full page, all sections, script tags
- `src/index.js`: JS entry point — all animation and interaction logic

**Configuration:**
- `tailwind.config.js`: Tailwind theme extensions (font family, custom height), content scanning glob
- `.stylelintrc`: Stylelint rules allowing Tailwind at-rules (`@tailwind`, `@apply`)
- `package.json`: `tailwind` build script and dependency declarations

**Core Logic:**
- `src/index.js`: Mobile nav toggle (lines 1–21), GSAP entrance timeline (lines 23–31), section scroll animations (lines 33–79)
- `src/index.css`: Tailwind directives (lines 3–6), custom `.custom-button` (lines 8–12), SVG keyframe animations (lines 14–207)

**Styles:**
- `src/index.css`: Authored CSS source — edit this
- `dist/main.css`: Compiled output — do not edit directly

**Assets:**
- `src/assets/img3.jpg`: About section profile image
- `src/assets/hero.svg` through `hero3.svg`: Hero illustration variants

## Naming Conventions

**Files:**
- Lowercase with no separators for source files: `index.html`, `index.css`, `index.js`
- Lowercase with no separators for config files: `tailwind.config.js`
- Kebab-case not currently used (no multi-word source files)

**Directories:**
- Lowercase short names: `src/`, `dist/`, `assets/`

**HTML IDs (section anchors):**
- Lowercase single words matching nav link hrefs: `#hero`, `#about`, `#skills`, `#projects`, `#contact`, `#navigation`

**HTML Classes (GSAP animation targets):**
- Kebab-case descriptive names: `.main-animation`, `.anime-text`, `.about-img`, `.about-text`, `.skill-title`, `.skill-single`, `.project-single`, `.contact-form`, `.contact-email`
- Mobile nav classes: `.mobile-menu`, `.mobile-menu-button`, `.menu-list`

**CSS Keyframe Names:**
- Kebab-case describing the motion: `main-slide`, `slide`, `wobble`, `hand-wobble`

**SVG Group IDs:**
- Lowercase with number suffixes for animated groups: `#main1`–`#main5`, `#line1`–`#line8`
- Descriptive PascalCase for character groups: `#Body`, `#hand`, `#rest`

## Where to Add New Code

**New Page Section:**
- HTML markup: Add a new `<div id="section-name" class="scroll-mt-12 ...">` block inside `index.html` before the `<!-- ! SCRIPTS -->` comment (line 1208)
- Navigation link: Add `<a href="#section-name">` to both the desktop menu (line 66–73) and the mobile menu `<ul>` (lines 94–130) in `index.html`
- Scroll animation: Add a new `gsap.timeline({ scrollTrigger: { trigger: ".section-container", start: "center bottom" } })` block in `src/index.js`

**New CSS Utility or Custom Style:**
- Edit `src/index.css` — add after the existing `@tailwind utilities` line or in the animations block
- Run `npm run tailwind` to recompile to `dist/main.css`

**New Static Asset:**
- Place image or SVG in `src/assets/`
- Reference from `index.html` using path `./src/assets/filename.ext`

**New JS Interaction:**
- Add to `src/index.js` — query DOM elements with `document.querySelector` or `document.querySelectorAll`
- GSAP and ScrollTrigger are available as globals (loaded from CDN before `src/index.js`)

## Special Directories

**`dist/`:**
- Purpose: Tailwind-compiled CSS output
- Generated: Yes — produced by `npm run tailwind`
- Committed: Yes (currently in repository)
- Do not hand-edit `dist/main.css`; always edit `src/index.css` and recompile

**`.planning/`:**
- Purpose: GSD planning and codebase analysis documents
- Generated: By GSD commands (`/gsd-map-codebase`, `/gsd-plan-phase`)
- Committed: Present in repository
