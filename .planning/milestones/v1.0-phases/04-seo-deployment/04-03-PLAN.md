    CREATE proxy.ts at project root (same directory level as app/, lib/, components/, next.config.mjs).

    Imports:
    - `import { NextResponse } from 'next/server'`
    - `import type { NextRequest } from 'next/server'`

    Named export `proxy` (NOT `middleware` — that name is deprecated in Next.js 16):
    ```
    export function proxy(request: NextRequest) { ... }
    ```

    Function body — three gates in order:
    1. Production guard (per D-11): `if (process.env.VERCEL_ENV !== 'production') { return NextResponse.next() }` — ensures preview and local environments are unaffected
    2. Hostname check (per D-10): `const host = request.headers.get('host') ?? ''` — MUST use `request.headers.get('host')` NOT `request.nextUrl.hostname` (Pitfall 4: nextUrl.hostname returns the deployment URL, not the browser's Host header)
    3. Redirect: `if (host === 'blog.yashkadam.com') { return NextResponse.redirect('https://yashkadam.com/blog', { status: 308 }) }`
    4. Default passthrough: `return NextResponse.next()`

    Exported `config` object with matcher (per Pitfall 6 — exclude static assets to avoid unnecessary proxy processing):
    ```
    export const config = {
      matcher: ['/((?!api|_next/static|_next/image|favicon.ico|sitemap.xml|robots.txt).*)'],
    }
    ```

    CRITICAL constraints:
    - Do NOT use `export function middleware(...)` — that is the deprecated Next.js 15 name
    - Do NOT set `runtime = 'edge'` — throws error in Next.js 16 proxy files
    - Do NOT use `request.nextUrl.hostname` for host detection — returns deployment URL, not browser Host header
    - Status code must be 308 (not 301 or 302) — 308 preserves the HTTP method on redirect (permanent, method-preserving)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Incoming Host header → redirect decision | The proxy reads `Host` header to determine if the request is from blog.yashkadam.com; spoofed Host headers could in theory trigger the redirect on a non-blog request |
| VERCEL_ENV → production guard | If VERCEL_ENV is incorrectly set to 'production' on a staging deploy, the redirect fires on preview URLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Spoofing | proxy.ts Host header check | accept | Host header spoofing on Vercel edge would require the request to actually route through Vercel's edge to the blog.yashkadam.com deployment — the redirect is harmless (sends to yashkadam.com/blog), no auth or data involved |
| T-04-07 | Denial of Service | proxy.ts running on every request | mitigate | config.matcher excludes _next/static, _next/image, and other static assets; VERCEL_ENV check short-circuits on non-production, reducing overhead |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; NextRequest and NextResponse are already bundled in next@16.2.6 |
</threat_model>
