## Milestone Summary

**Key Decisions:**

- Use `@next/mdx` exclusively — `next-mdx-remote` archived Feb 2026, `contentlayer` abandoned Jun 2023
- Start from v2 branch (Next.js 14 + Framer Motion); upgrade to Next.js 16.2.6
- Dark mode FOUC fix and `LazyMotion` pattern are Phase 1 — cannot be retrofitted safely
- Skills shown as tech stack tags on project cards — no separate skills section or progress bars
- Blog subdomain redirect requires `middleware.ts`, not Vercel dashboard (dashboard cannot redirect to a path)
- `rehype-pretty-code` 0.14.3 + Turbopack: Shiki themes must be passed as string names (not imported objects); `mdxRs: true` must be disabled
- `NEXT_PUBLIC_SITE_URL` set to `https://www.yashkadam.com` (with www) to match canonical Vercel URL
- Vercel Analytics placed outside `LazyMotion` boundary — packages carry their own `'use client'` directive
- `github` field on Project typed as `` `https://${string}` `` template literal for compile-time URL safety
- Brew Index GitHub link removed (confirmed 404); project card still shown without GitHub link

**Issues Resolved:**

- Dark mode FOUC (white flash on dark-preference browsers) — fixed via inline blocking script in root layout
- `rehype-pretty-code` ESM/Turbopack incompatibility — resolved by passing theme as string name
- `middleware.ts` subdomain redirect — production-gated via `VERCEL_ENV === 'production'` to avoid matching Vercel preview URLs
- Dead brew-index GitHub 404 — removed with conditional render guard

**Issues Deferred:**

- Vercel dashboard Analytics/Speed Insights toggle sign-off — human_needed flag; all code wired
- Phase verification files marked `human_needed` — documentation sign-offs, not unimplemented features (acknowledged at milestone close)

**Technical Debt Incurred:**

- `PORT-04` partially satisfied: Brew Index GitHub link removed (dead link); live demo link also absent. Project card still shown with description + tags only.
- No dark mode toggle in v1.0 — deferred to v2 (PLSH-01)
